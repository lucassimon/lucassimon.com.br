O ambiente de desenvolvimento

De acordo com o artigo fonte do criador e mantenedor do [loafer]() vamos criar nossa stack de desenvolvimento, simulando um ambiente AWS.

Primeiro baixe o container docker:

$ docker pull pafortin/goaws
$ docker run -d --name goaws -p 4100:4100 pafortin/goaws

Instale e realize as configurações do aws-cli:


pip install awscli

As configurações ficam em ~/.aws/config:

[default]
region = sa-east-1

E as credenciais em ~/.aws.credentials

[default]
aws_access_key_id = access_key_super_secret
aws_secret_access_key = access_key_here_be_a_dragon

Criando tópicos e filas no AWS local:

Através do ferramenta aws instalada e configurada previamente, vamos criar um tópico SNS chamado iugu-customer-created

$ aws --endpoint-url http://localhost:4100 sns create-topic --name iugu-customers-created
{
    "TopicArn": "arn:aws:sns:local-01:000000000000:iugu-customers-created"
}

E uma fila:

$ aws --endpoint-url http://localhost:4100 sqs create-queue --queue-name iugu-customers-main-prd.fifo
{
    "QueueUrl": "http://goaws:4100/queue/iugu-customers-main-prd.fifo"
}

E então inscrever o SNS criado na fila.

$ aws --endpoint-url http://localhost:4100 sns subscribe \ 
--topic-arn arn:aws:sns:local-01:000000000000:iugu-customers-created  --protocol sqs \
--notification-endpoint http://goaws:4100/queue/iugu-customers-main-prd.fifo

{
    "SubscriptionArn": "arn:aws:sns:local-01:000000000000:iugu-customers-created:04bdcef2-e2f6-4a8c-8ba6-f4b548832a37"
}
